磁盘大小

Filesystem Size Used Avail Use% Mounted on
devtmpfs 3.9G 0 3.9G 0% /dev
tmpfs 3.9G 0 3.9G 0% /dev/shm
tmpfs 3.9G 8.6M 3.9G 1% /run
tmpfs 3.9G 0 3.9G 0% /sys/fs/cgroup
/dev/vda1 40G 5.2G 33G 14% /
tmpfs 783M 0 783M 0% /run/user/0

  • 磁盘分为系统盘跟数据盘

  • 可以看到根路径 / 都是位于系统盘

  • /dev/vda - 系统盘，有一个分区vda1

  • /dev/vdb - 数据盘，有两个分区vdb1,vdb2

内存

  • 预留内存尽量在15%

  • swap 因为内存不够，使用部分磁盘空间来充当内存使用，虽然可以解决内存紧缺的问题，但是效率不高。尽量不要使用。

机器负载

load average: 0.07, 0.05, 0.05
1m 5m 15m

1分钟内的负载是0.07

  • 经验值： 生产上尽量控制在10， 否则服务器就认为卡

何时会出现机器负载为10这种可能，以及如何解决？

  • 计算程序： hive sql, spark, flink 密集计算，是不是要调优

  • 10min内很快启动了很多服务，那就不要短时间启动很多

  • 看是否被挖矿了， 某个进程cpu 100%

    yarn 软件
    redis软件
    默认端口号 会被扫描 进行注入 挖矿

  • 硬件问题 内存条损坏（最后一招，停掉服务，万能重启）

  • 有进程pid, 不一定有端口号

问题：ping ip 通， 端口号不通

可以查看配置文件是否设置为本地

[root@hadoop1 ~]# netstat -nlp|grep http
tcp6 0 0 :::80 :::* LISTEN 3478/httpd
tcp6 0 0 0.0.0.0:80 :::* LISTEN 3478/httpd
tcp6 0 0 192.168.1.101:80 :::* LISTEN 3478/httpd

80端口号服务可以对外
window或者其他服务器 可以ip+80 进行访问的

tcp6 0 0 localhost:80 :::* LISTEN 3478/httpd
tcp6 0 0 127.0.0.1:80 :::* LISTEN 3478/httpd
localhost 、127.0.0.1 代表本机
80端口仅限本机的其他服务可以访问，window或者其他服务器无法访问
