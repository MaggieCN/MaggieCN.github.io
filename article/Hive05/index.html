SQOOP

概念

  • 在关系型数据库和Hadoop之上建立了一个桥梁，让我们方便的完成数据的导入导出操作

  • Sqoop 有两个版本 ：1.4.7 Sqoop1 ；1.99.7 Sqoop2

    • V1 V2 没有任何关系 不兼容

  1. import

  • –table : emp

  • 导入路径 /user/hadoop/emp (当前是Hadoop用户所以在hadoop 下的表名)

这里可能会出现两个错误，如下：

  • 第一个错误 转自： https://blog.csdn.net/yu0_zhang0/article/details/79069251

  • 提示说我mysql数据中的emp表没有设置主键，解决方案a. --split-by; b. -m 1

  • Sqoop通可以过–split-by指定切分的字段，–m设置mapper的数量。-m 1 指定启动一个map进程，如果表很大，可以启动多个map进程，默认是4个

    1. split-by 根据不同的参数类型有不同的切分方法，如表共有100条数据其中id为int类型，并且我们指定–split-by id，我们不设置map数量使用默认的为四个，首先Sqoop会取获取切分字段的MIN()和MAX()即（–split -by），再根据map数量进行划分，这是字段值就会分为四个map：（1-25）（26-50）（51-75）（75-100）。
    2. 根据MIN和MAX不同的类型采用不同的切分方式支持有Date,Text,Float,Integer， Boolean,NText,BigDecimal等等。
    3. 所以，若导入的表中没有主键，将-m 设置称1或者设置split-by，即只有一个map运行，缺点是不能并行map录入数据。（注意，当-m 设置的值大于1时，split-by必须设置字段） 。
    4. split-by即便是int型，若不是连续有规律递增的话，各个map分配的数据是不均衡的，可能会有些map很忙，有些map几乎没有数据处理的情况。

作业：

分组TopN
窗口函数

city_info.sql product_info.sql user_click.txt

城市表：MySQL
商品表：MySQL
用户行为日志：Hive

按照区域求最受欢迎商品的TopN

思路：将城市表与商品表用sqoop抽取到hive中，用户行为日志load到hive中

第一步在hive中建表

create table city_info(
city_id int,
city_name string,
area string
)
row format delimited fields terminated by ‘\t’;

create table product_info(
product_id int,
product_name string,
extend_info string
)
row format delimited fields terminated by ‘\t’;

create table user_click(
user_id string,
session_id string,
time string,
city_id int,
product_id int
) partitioned by(day string)
row format delimited fields terminated by ‘,’;

第二步sqoop抽取数据到hive

第三步：分成多步，落中间表
select * from user_click where day=‘2016-05-05’;
user_click: product_id user_id city_id

==> product点击基本信息信息
drop table tmp_product_click_basic_info;
create table tmp_product_click_basic_info as
select
u.product_id, u.city_id, c.city_name, c.area
from
(select product_id, city_id from user_click where day=‘2016-05-05’) u
join
(select city_id, city_name, area from city_info) c
on u.city_id = c.city_id;

==> 区域下的product的点击次数

drop table tmp_area_product_click_cnt;
create table tmp_area_product_click_cnt as
select
product_id, area, count(1) click_cnt
from
tmp_product_click_basic_info
group by product_id, area;
select * from tmp_area_product_click_cnt limit 5;

==> 获取完整的商品点击次数信息
drop table tmp_area_product_full_click_cnt;
create table tmp_area_product_full_click_cnt as
select
a.product_id, b.product_name, a.click_cnt, a.area,
if(get_json_object(extend_info,’$.product_status’) ==‘0’,‘self’,‘third’) product_status
from tmp_area_product_click_cnt a join product_info b
on a.product_id = b.product_id;
select * from tmp_area_product_full_click_cnt limit 5;

==> 分组的TopN

select * from
(
select
product_id,product_name,click_cnt,area,product_status,
row_number() over(partition by area order by click_cnt desc) rank
from
tmp_area_product_full_click_cnt) t
where t.rank <= 3;

作业：分组TopN的结果export到MySQL

  • 一定要定义row format 的\t as select 的直接在create和as select 中间定义
  • –direct 报 mysqldump 错误，直接去掉即可（解决mysqldump再更新）
  • 导出的时候一定要先建mysql表，并且定义的字段类型要与hive中的一致
