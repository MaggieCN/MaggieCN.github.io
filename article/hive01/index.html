The Apache Hive ™ data warehouse software facilitates reading, writing, and managing large datasets residing in distributed storage using SQL. Structure can be projected onto data already in storage. A command line tool and JDBC driver are provided to connect users to Hive.

Hive 架构

Hive是构建在hadoop之上的数据仓库（狭义）——HDFS\ YARN\ MR

hdfs： hive的数据存放在分布式存储中（HDFS）, 元数据是存在metastore 底层数据库中（mysql）

mr: hive作业（sql）通过hive的框架翻译成mr作业

目前mr已经被淘汰了， tez\spark

不管底层到底是使用什么引擎，对于用户来说是不感知

同样的sql, 只需要通过一个参数切换，就可以运行不同的执行引擎之上

hadoop: 开发可以使用单机，生产上肯定是分布式

hive没有集群的概念，就是个客户端（就是个提交机器）-作业都在yarn进行

yarn: hive 的作业是提交到yarn上运行

hive职责：将sql 翻译成底层对应的执行引擎作业

统一的元数据管理（metadata）== metastore

元数据概念：描述数据的数据 schema

不能停留在应用层面上

生产上需要注意：

需要考虑-HA: 高可用

mysql 不具备高可用

至少要有两个mysql

active down 掉- 切到standby 里

hive & rdbms(关系型数据库)

都是支持分布式的

不同：

节点数：hive 多节点

成本： hive低（廉价机器上）

数据量：

hive 支持update 但是0.14 版本以后开始支持，而且生产上离线数据不建议使用

事务：都支持，但是大数据没有实际性用途

延时性： mysql 延时短/ 离线作业（处理）时间长

hive 适用场景：

批处理、离线处理，延时性大，尽量少涉及到update/delete

hive 优点：

容易上手、比mr简单的多，写sql就可以

缺点：高延时

hive的执行速度vs mysql 执行速度谁快谁慢？

hive sql / mysql sql 有啥关系

hive是建立在mysql之上的
